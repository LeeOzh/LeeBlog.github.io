###第一天

javascript:

JS数据类型
  • 基本数据类型 （值类型）
    • 字符串：string
    • 数值：number
    • 布尔值：true | false
    • null : null (null的类型为obje，是一个遗留的bug)
      • 变量的值不会主动为null，除非人为设置为null
    • undefined
      • 声明变量但没有给变量赋值，该变量的值为undefined
  • 复杂数据类型（引用类型）
    • 数组（Array）
    • 对象 （Object）
    • 函数（Function）
    • Error对象
    • Math对象

    • 存储的是具体的值
  • 引用类型:对象类型(Object Function Date Math String Number)
    • 存储的是地址（引用），通过该引用可以找到对应的数据

js相等操作符

  • = 赋值

函数的相关知识点

函数的创建
函数的参数
  • 形式参数（形参），形参相当于是函数体内声明的局部变量，命名无意义
    • 形参叫声明名字都可以，作用是占位和接受实参
  • 实际参数（实参），函数调用的时候，传递给函数的具体的数据
  • 函数在JavaScript中是一等公民
    • 在代码中凡是对象可以出现的地方（变量 | 函数参数 | 函数的返回值），函数都可以出现
    • 专业术语：高阶函数
      • 如果一个函数的参数是参数或者返回值是函数，那么这样的函数就被叫做高阶函数
    • 专业术语：回调函数
      • 如果一个函数作为其他函数的参数，那么该函数就被叫做回调函数
函数的调用
  • 语法：函数名（）；
  • 关键：把实参赋值给形参，执行函数体
函数总结

  • 如果函数的参数是值类型的数据，那么在函数内部修改形参不会影响到实参的值

  • 如果函数的参数是引用类型的数据，那么函数内部修改形参会影响刀实参的值，他们是共享的。

JS对象的动态特性
  • JavaScript语言支持动态的设置（增加|删除|查询|修改）对象的内容（属性|方法）
  • JavaScript语言很灵活
  • 补充点：在window全局对象中存在默认的name属性，是一个空字符串
  • 访问（查询）对象的成员
    • 点语法：对象.需要访问的key
    • 中括号：对象[‘key’];
  • 修改对象的成员
    • 给对象成员赋值

break和continue

面向过程和面向对象编程

  • 面向过程和面向对象都是解决问题的一种方式
面向过程
  • 面向过程是把要解决的问题拆分成一个个的具体的步骤，严格按顺序来处理(函数)
  • 面向过程关注点：
    • 解决问题的每一个步骤
面向对象

  • 面向对象是找到合适的对象来解决问题

  • 面向对象是对面向过程的封装

  • 面向对象的关注点

    • 解决问题需要的对象

  • 面向对象变成相关概念：

    • 具体的对象
    • 对象都有什么
    • 面向对象编程：合适的对象

  • 面向对象的三大特性：

    • 封装
    • 继承
    • 多态
面向对象的基本概念
  • 成员：对象的属性和方法
  • 实例成员：实例对象的属性和方法
  • 静态成员：构造函数自己的属性和方法
  • 原型成员：构造函数对应的原型对象的属性和方法
  • 私有成员：构造函数中声明的变量和函数，因为他们只能再函数内部被访问，随意是私有的
  • 特权方法：在函数内部使用（访问—读取+设置）私有成员的实例被称为是特权方法

#####创建对象的4种方式

  • 用字面量方式创建对象

    • var obj = {

      }

    • 缺点：如果批量创建大量相似的对象，会很麻烦（代码重复）

  • 内置构造函数创建对象

    • var obj = new Object( );

  • 简单工厂函数创建对象

  • 自定构造函数创建对象

    • 自定构造函数：函数名默认大写字母开头

    • 自定义构造函数创建对象过程中的执行细节

      • 默认会创建一个空对象
      • 默认把创建好的空对象赋值给this
      • 默认设置this的原型对象为当前构造函数的原型对象
      • 通过this来添加属性和方法
      • 默认会把内部新创建的对象返回 return this

  • 自定构造函数和工厂函数创建对象的对比

    • 构造函数的首字母要求大写
    • 在使用函数创建对象的时候，需要使用new来调用
    • 在函数内部不需要再主动的创建Object类型的对象
    • 在函数内部不要主动的把创建好的对象return

  • 自定义构造函数创建对象的返回值情况

    • 默认情况，返回的内部新创建的实例对象（this)
    • 主动添加return
      • 如果返回的是值类型的数据，那么会被忽略，真正返回的还是this
      • 如果返回的是引用类型的数据，那么就替换this直接返回
        • 例: return [1,2,3,4];

  • 自定义构造函数创建对象存在的问题

    • 如果创建的对象非常多，而函数体是一样的，就会造成资源的浪费

  • 构造函数知识点：

    • 构造器属性 （constructor）

      • 作用：能通过该属性得到当前对象的类型（构造函数的名称）

      • 该属性只想的是创建当前对象的构造函数

  • 构造函数的调用和命名

    • 构造函数本身和普通的函数没有区别
    • 为了认为的区分构造函数和普通的函数，约定构造函数的首字母大写
    • 构造函数可以像普通函数一样来进行调用，此时内部的this指向window
    • 重点：构造函数需要使用new来调用
      • new关键字的作用：创建新的对象并且赋值给this；
this
  • this：指针，总是指向一个对象（引用类型）
  • 在全局作用指向window
  • 在函数里的this
    • this指向谁由函数的调用方式来决定
    • 普通函数调用 fn( ) ; this 指向 window
    • 对象的方法调用 obj.fn( ) ; this 指向 obj
    • 构造函数调用 new CreatePerson( ) ; this 指向内部新创建的实例对象
    • 函数上下文调用 call | apply ； this 指向call方法或者是apply方法的第一个参数
原型对象

  • 每一个构造函数都有一个与之相互关联的对象，该对象被称为原型对象

  • 默认的原型对象是一个Object类型的空对象（拥有constructor属性）

  • 访问原型对象

    • 构造函数：构造函数.prototype
    • 实例对象：实例对象. _ _proto _ _

  • 设置原型对象：

    • 构造函数的原型对象替换为一个新创建的对象

实例对象和实例化过程

  • 构造函数：Person

  • 构造函数的原型对象：Person.prototype
  • 原型对象（创建该实例对象的构造函数向关联的原型对象）P1的原型对象
  • 实例对象：使用new来调用构造函数创建出来的对象就是实例对象
  • 实例化：构造函数创建实例对象的过程叫做实例化
  • 对象的成员：对象的属性 + 对象的方法
  • 实例成员:实例对象身上的属性和方法,实例属性 或者是实例方法 name age showInfo()
  • 原型成员:对应的原型对象身上的成员,原型属性和原型方法 color showColor()
  • 静态成员:直接添加在构造函数自己身上的属性和方法
    • 注意点：
      • 静态成员只有构造函数可以直接访问，实例对象无法访问
      • 构造函数无法访问实例对象的成员
关键字

  • 检查对象中是否存在指定的属性

    • in关键字 实例成员 && 原型成员
    • hasOwnProperty（） 实例成员

  • 判断某个对象是否是指定实例对象的原型对象

    • isPrototypeOf( )

    • instanceof

    • 作用相同

原型链的结构
  • 每一个实例对象都是由构造函数创建出来的
  • 每一个构造函数都有默认关联的原型对象
  • 原型对象本身也是对象，所以它也有自己的构造函数
  • 原型对象的构造函数也由默认关联的原型对象
    • 以上就构成了一种链式的访问结构，称为原型链
    • Object.prototype是原型链的终点
面向对象继承的实现方式

  • 混入式继承（拷贝 / 复制）

    • 存在的问题：如果拷贝的对象中属性对应的是引用类型，那么会存在共享问题

  • 原型式继承 （能够访问父类的原型对象中的原型成员）

  • 扩展内置对象

    • 问题：

      • 多人开发的时候可能会出现一些奇怪的BUG （譬如使用for…in循环遍历数组）
      • 在访问属性和方法的时候，会搜索（查找–遍历），如果增加太多的方法，在访问成员的时候性能会降低。

    • 安全的扩展内置对象

  • 原型链继承

    • 使子（构造函数）的原型对象设置为父（构造函数）的实例对象，就可以访问父(构造函数)原型对象的成员和方法；
      • 可以获取原型成员 + 实例成员（由于父构造函数无法传参，实例成员被写死）
    • 注意点：
      • 设置原型链继承的时候，先完成继承，再添加原型成员
      • 修正构造器属性
    • 问题：
      • 无法向构造函数传递参数
      • 共享的问题
      • 搜索更长

  • 组合继承（原型式继承 + 原型链继承）（访问实例对象成员 + 访问父类原型对象成员）

call,apply

  • 作用

    • 借用其他对象的方法
    • 设置函数（方法）中this的指向，指向的使call和apply方法的第一个参数

  • 语法：

    • call
      • 对象1.方法.call（对象2，参数1，参数2，参数3）
      • 对象2后面的参数为可选参数
    • apply
      • 对象1.方法.call（对象2，[参数1，参数2，参数3]）
      • 对象2后面的参数需要用数组来传递。

  • 来源：

    • call和apply方法存在于函数对象的原型对象上面

  • 区别：

    • 传递参数的方式不一样
    • 形参的个数 call：1；apply：2；

Object的静态成员
  • create（）创建独享并且设置原型对象
  • assign（）合并对象：ES6的方法，合并的时候如果存在同名的成员，后面的会覆盖前面

惰性函数

###第六天

闭包
  • 简单理解：
    • 闭包就是能够读取其他函数内部变量的函数
    • 在JavaScript语言中，只有函数内部的子函数才能读取局部变量，因此可以把闭包简单理解成 “定义在一个函数内部的函数”；
    • 本质上，闭包就是将函数内部和函数外部连接起来的一座桥梁
  • 闭包的用途：
    • 可以读取函数内部的变量
    • 让函数内部的变量始终保存在内存中
